# 2026-02-19: 반복문 문법 유연성 개선 (Colons & Implicit Keywords)

## 선물 내역
- **반복문 콜론(:) 허용**: Python 등 다른 언어에 익숙한 정한을 위해 반복문 끝에 콜론을 붙여도 정상 작동하도록 개선했습니다.
- **반복문 키워드 생략 허용**: `목록의 요소마다 반복` 대신 `목록의 요소마다` 만으로도 반복문이 실행되도록 문법을 확장했습니다.
- **조사 분리 지능화**: `목록의`, `요소마다` 처럼 조사가 붙은 경우에도 파서가 똑똑하게 분리하여 인식합니다.
- **실행 순서 최적화**: 반복문 내부에서 현재 요소의 값이 루프 바디가 실행되기 전에 먼저 변수에 할당되도록 수정하여 안정성을 높였습니다.

## 기술적 변경사항
- `core/prepare/parse/rule/list-loop.ts`, `count-loop.ts`: 콜론(`:`)이 포함된 패턴 추가.
- `core/prepare/parse/split-variable-name.ts`: `의`, `마다` 조사 분리 로직 강화.
- `core/node/listLoop.ts`: 변수 할당(`setVariable`)과 실행 보고(`onRunChild`) 순서 조정.
- `core/constant/reserved-words.ts`: `마다`를 예약어에 추가하여 파싱 일관성 확보.
